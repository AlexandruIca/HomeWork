bool globals::g_inside_draw_call {false}

This variable is needed to see whether the user wants to draw a static primitve or not. By static primitive I mean this:

int main()
{
triangle(/*parameters...*/);
return draw();
}

This will draw a static triangle since it's called before the draw call. The function will add the Triangle shape to the global vector of shapes.

But if it's called like this:

int main()
{
return draw(WITH{
triangle(/*parameters...*/);
});
}

This will add nothing to the global vector of shapes. It's just calling SDL_RenderDrawTriangle. Keep in mind this is available for all primitives, not just triangles, this is just an example. This behaviour allows for games like snake where I need to draw shapes on the fly, I cannot just declare a lot of Rectangle shapes because a shape's destructor is non trivial and it adds shapes to the global vector.
